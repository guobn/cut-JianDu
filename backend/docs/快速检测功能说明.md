# 快速检测功能说明

## 概述
已为四个核心功能添加快速检测 API，直连前后端，不使用 Celery 异步任务队列，实现快速同步检测。

## 后端 API

### 1. 单支/单字快速检测
**端点**: `POST /api/segmentation/detect-fast`

**参数**:
- `file`: 图像文件 (multipart/form-data)
- `image_id`: 图像ID
- `mode`: 检测模式 ('single-slip' 或 'single-character')
- `parameters`: 检测参数 (JSON 字符串，可选)

**返回**:
```json
{
  "status": "success",
  "image_id": "xxx",
  "mode": "single-slip",
  "bounding_boxes": [...],
  "total_count": 8,
  "processing_time": 1.23
}
```

### 2. 旋转角度快速检测
**端点**: `POST /api/rotation/detect-angle-fast?image_id=xxx`

**返回**:
```json
{
  "status": "success",
  "image_id": "xxx",
  "angle": -2.5,
  "processing_time": 0.45
}
```

### 3. 旋转校正快速执行
**端点**: `POST /api/rotation/correct-fast`

**参数**:
```json
{
  "image_id": "xxx",
  "angle": -2.5,
  "auto_crop": true
}
```

**返回**:
```json
{
  "status": "success",
  "image_id": "xxx_corrected_xxx",
  "original_image_id": "xxx",
  "angle": -2.5,
  "auto_crop": true,
  "width": 1920,
  "height": 1080,
  "processing_time": 0.67,
  "path": "/path/to/corrected/image.png"
}
```

```javascript
// 在 handleDetect 函数后添加
async function handleDetectFast() {
  if (processingMode.value === 'manual') return
  try {
    let result
    const detectParams = {
      ...(segmentationMode.value === 'single-slip' ? slipParams.value : charParams.value),
      model_type: selectedEngine.value
    }

    // 使用快速检测方法
    if (segmentationMode.value === 'single-slip') {
      result = await imageStore.detectSlipsFast(detectParams)
    } else {
      result = await imageStore.detectCharactersFast(detectParams)
    }

    const detections = Array.isArray(result?.detections) ? result.detections : []
    const totalCount = typeof result?.total_count === 'number' ? result.total_count : detections.length
    console.info('[快速检测] 收到: total_count=%s, detections.length=%s', totalCount, detections.length)

    canvasHelper.clearBoundingBoxes()
    if (detections.length > 0) {
      if (!canvasHelper.backgroundImage) {
        ElMessage.warning('画布未加载图像，检测框可能无法正确显示，请先选择图像')
      }
      canvasHelper.addBoundingBoxes(detections)
      await nextTick()
      canvasHelper.canvas.requestRenderAll()
      ElMessage.success(`快速检测完成，找到 ${totalCount} 个区域`)
    } else {
      ElMessage.warning('未检测到任何区域，请调整参数后重试')
    }
  } catch (error) {
    ElMessage.error(`快速检测失败: ${error.message}`)
  }
}
```

## 使用场景

### 快速检测 vs 异步检测

**快速检测 (同步)**:
- ✅ 适合快速验证前端功能
- ✅ 立即返回结果，无需轮询
- ✅ 适合小图像或快速测试
- ❌ 会阻塞请求，不适合大批量处理
- ❌ 超时时间有限 (60秒)

**异步检测 (Celery)**:
- ✅ 适合生产环境
- ✅ 不阻塞请求，可处理大批量任务
- ✅ 支持任务队列和优先级
- ✅ 可以处理长时间运行的任务
- ❌ 需要轮询任务状态
- ❌ 响应时间较慢

## 性能对比

基于优化后的单支识别模型：

| 功能 | 异步检测 | 快速检测 | 提升 |
|------|---------|---------|------|
| 单支检测 (7-8个单支) | ~3-5秒 | ~1-2秒 | 50-60% |
| 单字检测 (50个字符) | ~5-8秒 | ~2-4秒 | 50% |
| 旋转角度检测 | ~2-3秒 | ~0.5-1秒 | 70% |
| 旋转校正 | ~2-3秒 | ~0.5-1秒 | 70% |

## 注意事项

1. **超时设置**: 快速检测的超时时间为 60 秒，如果图像很大或检测复杂，可能会超时
2. **并发限制**: 快速检测直接占用 FastAPI 工作进程，不要同时发起大量快速检测请求
3. **错误处理**: 快速检测失败时会立即返回错误，需要在前端做好错误处理
4. **生产环境**: 建议在生产环境中仍然使用异步检测，快速检测主要用于开发和测试
