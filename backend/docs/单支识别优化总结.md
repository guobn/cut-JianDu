# 单支识别优化总结

## 问题描述
一张图片本来有7-8个单支，但只能识别出一两个。单支之间一般没有重叠区域，且单支和背景的差异比较大，理论上应该是一个简单的任务。

## 根本原因分析
通过代码审查发现，单支检测使用 YOLOv8 模型，但后处理过滤条件过于严格，导致大量有效检测框被过滤掉：

1. **置信度阈值过高**: 0.25 可能过滤掉一些有效的单支
2. **长宽比范围过严**: [0.02, 0.6] 不适合所有单支形状
3. **最小尺寸要求过严**: min_w=20, min_h=40 过滤掉较小的单支
4. **边界 margin 过滤**: margin=5 过滤掉靠近边缘的单支

## 优化方案

### 1. 降低置信度阈值
```python
# 优化前
conf_threshold = 0.25

# 优化后
conf_threshold = 0.15  # 降低 40%，提高召回率
```

### 2. 放宽长宽比范围
```python
# 优化前
ar_min = 0.02
ar_max = 0.6

# 优化后
ar_min = 0.01  # 降低 50%
ar_max = 0.8   # 提高 33%
```

### 3. 降低最小尺寸要求
```python
# 优化前
min_w = 20
min_h = 40

# 优化后
min_w = 10  # 降低 50%
min_h = 30  # 降低 25%
```

### 4. 减小边界 margin
```python
# 优化前
margin = 5

# 优化后
margin = 2  # 降低 60%
```

### 5. 添加详细的过滤日志
新增过滤统计功能，记录每种过滤条件过滤掉的框数量：
- class: 类别不匹配
- aspect_ratio: 长宽比超出范围
- size: 尺寸太小
- margin: 超出边界
- invalid: 无效框（宽高 <= 0）

## 修改的文件

1. **app/services/segmentation_service.py**
   - 修改 `_detect_single_slips_yolo` 函数
   - 优化过滤参数
   - 添加详细的过滤日志

2. **app/config.py**
   - 更新默认置信度阈值: 0.25 → 0.15

3. **.env**
   - 更新环境变量: YOLOV8_SLIP_CONF_THRESHOLD=0.15

## 预期效果

优化后，单支检测的召回率应该显著提高：
- **置信度降低 40%**: 可以检测到更多低置信度但有效的单支
- **长宽比范围扩大**: 适应更多形状的单支
- **最小尺寸降低**: 可以检测到较小的单支
- **边界 margin 减小**: 可以检测到靠近边缘的单支

预计从原来的 1-2 个检测框提升到 5-8 个检测框（接近实际单支数量）。

## 进一步优化建议

如果优化后效果仍不理想，可以考虑：

### 1. 进一步降低置信度阈值
```python
# 在 .env 中设置
YOLOV8_SLIP_CONF_THRESHOLD=0.1  # 或更低
```

### 2. 完全移除边界 margin 过滤
```python
# 在 segmentation_service.py 中
margin = 0  # 不过滤边界框
```

### 3. 调整长宽比范围
根据实际单支的形状特征，进一步调整：
```python
ar_min = 0.005  # 更窄的单支
ar_max = 1.0    # 更宽的单支
```

### 4. 使用 SAHI 切片推理
对于大图，可以启用 SAHI 切片推理，提高小目标检测能力：
```python
# 在配置中启用
sahi_slice_size = 640
sahi_overlap_ratio = 0.25
```

### 5. 重新训练模型
如果过滤条件放宽后出现大量误检，建议：
- 收集更多训练数据
- 重新训练 YOLOv8 模型
- 使用更大的模型（如 yolov8x.pt）

## 监控指标

优化后需要关注以下指标：
1. **召回率**: 检测到的单支数量 / 实际单支数量
2. **精确率**: 正确检测的单支数量 / 检测到的单支数量
3. **过滤统计**: 各过滤条件过滤掉的框数量
4. **处理时间**: 确保优化不影响性能
